# vue 内置指令

```
1) v-text // v-text主要用来更新textContent，可以等同于JS的text属性
例：<span v-text='msg'></span> 等价 <span>{{ msg }}</span>

2) v-html // 双大括号的方式会将数据解释为纯文本，而非HTML。为了输出真正的HTML，可以用v-html指令。它等同于JS的innerHtml属性。
例：<div v-html='rawHtml'></div>

3) v-pre // v-pre主要用来跳过这个元素和它的子元素编译过程。可以用来显示原始的Mustache标签。跳过大量没有指令的节点加快编译。
例：<div id="app">
        <span v-pre>{{message}}</span>  //这条语句不进行编译
        <span>{{message}}</span>
    </div>
最终仅显示第二个span的内容

4) v-cloak // 这个指令是用来保持在元素上直到关联实例结束时进行编译。
例：<span v-cloak>{{name}}</span>
    [v-cloak] {
      display: none;
    }
{{name}}这个内容，在网速很慢的情况下，一开始会直接显示{{name}}这个内容，等网络加载完成了，才会显示smyhvae。那这个闪烁的问题该怎么解决呢？
解决办法是：通过v-cloak隐藏{{name}}这个内容，当加载完毕后，再显示出来。

5) v-once // v-once关联的实例，只会渲染一次。之后的重新渲染，实例极其所有的子节点将被视为静态内容跳过，这可以用于优化更新性能。
例：<div v-once>//有子元素
        <h1>comment</h1>
        <p>{{msg}}</p>
    </div>
    <my-component v-once:comment="msg"></my-component>  //组件

6) v-if
7) v-else
8) v-else-if
9) v-show
10) v-for
11) v-bind 简写 :
12) v-model
      .lazy：默认情况下，v-model同步输入框的值和数据。可以通过这个修饰符，转变为在change事件再同步。
      .number：自动将用户的输入值转化为数值类型
      .trim：自动过滤用户输入的首尾空格
13) v-on 简写 @
      .stop：阻止事件继续传播
      .prevent：事件不再重载页面
      .capture：使用事件捕获模式,即元素自身触发的事件先在此处处理，然后才交由内部元素进行处理
      .self：只当在 event.target 是当前元素自身时触发处理函数
      .once：事件将只会触发一次
      .passive：告诉浏览器你不想阻止事件的默认行为
```
